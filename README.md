## 📖 Історія проекту

Цей проект народився не просто як програмне забезпечення, а як рішення реальної проблеми в українській школі під час війни.

Мій батько працює вчителем фізики. Він завжди прагнув до автоматизації та інновацій, тому спочатку намагався налаштувати систему шкільних дзвінків через розумні реле Sonoff. Це працювало, але реальність внесла свої корективи. Повітряні тривоги, постійні перегляди розкладів через відключення світла та необхідність оперативно змінювати час уроків зробили використання звичайних IoT-рішень занадто складним. Систему доводилося постійно вимикати, і вчителям знову доводилося давати дзвінки вручну.

Тоді ми вирішили створити **School Bell** — спеціалізовану програму, яка дозволяє гнучко керувати розкладом уроків та автоматично подавати дзвінки через реле, підключене до комп'ютера. Навіть у найскладніші часи технології мають допомагати освіті залишатися організованою.

## 🚀 Основні можливості

- **Гнучке керування розкладами:** Створення кількох варіантів розкладів (стандартний, скорочений, святковий).
- **База даних:** Збереження інформації про вчителів, предмети, класи та аудиторії за допомогою SQLite.
- **Апаратна інтеграція:** Керування фізичним шкільним дзвінком через USB/WIFI-реле.
- **Автоматизація тривог:** Моніторинг повітряних тривог та автоматичне подання відповідних сигналів.
- **Веб-табло:** Трансляція розкладу уроків на телевізори або монітори в локальній мережі (за замовчуванням на порту **9090**).
- **Преміальний дизайн:** Сучасний "Modern Academic Soft UI" для комфортної щоденної роботи.

## 🔌 Підтримуване обладнання

Програма підтримує два типи пристроїв для керування фізичним дзвінком: USB-реле (пряме підключення до ПК) та Wi-Fi-реле Shelly (керування через локальну мережу).

### 1. USB-реле (HID)

Працює як USB HID-пристрій та не потребує встановлення додаткових драйверів у Windows.

| Модель / Чіп | Опис | Статус |
| :--- | :--- | :---: |
| **USBRelay2** | VID_16C0 PID_05DF (2 канали) | ✅ Протестовано |
| **USBRelay1/4/8** | Моделі на 1, 4 або 8 каналів | ℹ️ Теоретично сумісно |
| **Сумісні USB HID-реле** | Пристрої на базі VID_16C0 / PID_05DF | ℹ️ Теоретично сумісно |

### 2. Wi-Fi-реле (Shelly)

Керування здійснюється через локальну мережу за допомогою HTTP RPC API.

| Покоління | Пристрої | Статус |
| :--- | :--- | :---: |
| **Gen3** | Shelly 1 Mini Gen3 та інші пристрої Gen3 | ✅ Протестовано |
| **Gen2** | Shelly Plus 1, Shelly Pro 1 та інші пристрої Gen2 | ℹ️ Теоретично сумісно |
| **Gen1** | Shelly 1, Shelly 1PM та інші пристрої Gen1 | ℹ️ Теоретично сумісно |

> **Рекомендація:** Для стабільної роботи Shelly рекомендується налаштувати постійну IP-адресу (DHCP Reservation) на роутері.

## 📦 Встановлення та дані користувача

Програма постачається у вигляді зручного інсталятора Windows:

1. **Встановлення:** Запустіть файл [SchoolBell_Setup_v1.0.0.exe](https://github.com/Asic95/school_bell/releases/download/v1.0.0/SchoolBell_Setup_v1.0.0.exe) та дотримуйтесь інструкцій (доступна українська мова).
2. **Дані користувача:** Програма зберігає всі ваші розклади та налаштування у папці `%APPDATA%\SchoolBell\`. Це означає, що при оновленні програми або її перевстановленні ваші дані **не будуть втрачені**.
3. **Резервні копії:** Рекомендуємо періодично використовувати функцію "Створити копію" у розділі "Система" для додаткової безпеки ваших даних.
