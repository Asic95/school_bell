# 🗺 Огляд системи School Bell

Цей документ містить опис поточної структури проекту, функціональних можливостей та взаємозв'язків між модулями. Він служить довідником для розробки та підтримки системи.

## 🏗 Архітектура
- **Мова:** Java 21
- **UI Фреймворк:** JavaFX (Дизайн-система **Modern Academic Soft UI v2**)
- **База даних:** SQLite (`school_bell.db`)
- **Збірка:** Maven
- **Веб-компонент:** Вбудований сервер для трансляції розкладу (теми `classic`, `modern`, `cyber`).

## 📂 Структура папок та ключові класи

### 🖥 Інтерфейс Користувача (`com.schoolbell.ui`)
Система використовує централізовану фабрику компонентів для забезпечення візуальної цілісності.

- **`UIStyles` / `ControlFactory`**: Фундамент дизайну. Реалізовано "Glassmorphism-lite" поверхні (28px радіус, м'які тіні), Indigo-акценти та преміальну типографіку.
    - `MODERN_CHECKBOX_STYLE`: Кастомні закруглені чекбокси з Indigo-заливкою.
    - `HEADER_STYLE`: Посилені заголовки (14px, weight 900, 1.5px letter-spacing).
    - `createTimeCombo`: Спеціалізовані випадаючі списки часу з авто-форматуванням `%02d`.
- **`DashboardView`**: Головний екран. 
    - **Smart Day Logic**: Коректне відображення стану "Очікування" до початку першого уроку.
    - **Contextual Relay Status**: Явне відображення "Режиму симуляції" з кольоровим кодуванням.
- **`SystemView`**: Конфігурація системи.
    - **Reactive Journal**: Новий компонент `SystemJournalPane` з'являється автоматично при активації режиму симуляції.
    - **Setting Tiles**: Організація параметрів у плаваючі картки преміум-класу.
- **`EfirView`**: Центр трансляції.
    - **Stable Layout**: Права колонка пристроїв зафіксована на 480px для запобігання layout-shifting.
    - **Device Monitoring**: Покращені `DeviceRow` з трьохрядковим виводом інформації (Назва, IP, Статус).
- **`AnnouncementEditorDialog`**: Повністю перероблений інтерфейс створення оголошень з використанням уніфікованих полів вводу та покращеним фокусуванням текстових областей.

### 🧠 Сервіси (`com.schoolbell.service`)
- **`SignalService`**: Керування реле. Підтримка симуляції та "Хвилини мовчання".
- **`AnnouncementService`**: Керування базою активних та архівованих оголошень.
- **`ConfigService`**: Глобальне сховище налаштувань, включаючи параметри дизайну та мережі.
- **`DatabaseManager`**: Керує SQLite, підтримує міграції схеми даних.

### 🔌 Апаратна частина (`com.schoolbell.hardware`)
- **`RelayController`**: Розумний драйвер USB-реле. Автоматично перемикається в режим логування при увімкненні симуляції.

## 🛠 Поточний функціонал
1.  **Преміальний Soft UI:** Використання сучасних SaaS-патернів: великі відступи (28px), закруглені кути, чітка ієрархія заголовків.
2.  **Інтелектуальний розклад:** Автоматичне визначення поточного контексту (урок/перерва/очікування/кінець дня).
3.  **Динамічне логування:** Візуальний журнал подій, який інтегрується в інтерфейс саме тоді, коли це необхідно для налаштування.
4.  **Повний контроль ефіру:** Моніторинг підключених табло, керування темами оформлення та миттєва перевірка доступності портів у Firewall.
5.  **Адаптивність вводу:** Використання спеціалізованих ComboBox для часу та DatePicker з сучасним стайлінгом для запобігання помилкам вводу.
