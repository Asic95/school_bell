# 🗺 Огляд системи School Bell

Цей документ містить опис поточної структури проекту, функціональних можливостей та взаємозв'язків між модулями. Він служить довідником для розробки та підтримки системи.

## 🏗 Архітектура
- **Мова:** Java 21
- **UI Фреймворк:** JavaFX (з використанням Soft UI/Material 3 дизайну)
- **База даних:** SQLite (`school_bell.db`)
- **Збірка:** Maven
- **Веб-компонент:** Вбудований сервер для трансляції розкладу (підтримка тем `classic`, `modern` та нової висококонтрастної `cyber` через WebSockets).

## 📂 Структура папок та ключові класи

### 🧠 Сервіси (`com.schoolbell.service`)
Це "мозок" програми, де зосереджена вся бізнес-логіка:
- **`DatabaseManager`**: Керує підключенням до SQLite та міграціями (підтримує оновлення схеми для кабінетів).
- **`ScheduleService`**: Логіка розкладу дзвінків, визначення поточного уроку/перерви.
- **`SignalService`**: Керування подачею сигналів (дзвінків) через реле. Включає режим "Хвилина мовчання" о 09:00 та інструменти діагностики обладнання.
- **`AcademicService`**: Управління даними про вчителів, предмети, класи, аудиторії та заміни.
- **`AudioService`**: Відтворення аудіо-файлів (мелодій дзвінків).
- **`BroadcastService`**: Керування трансляцією даних на зовнішні екрани/веб-панелі.
- **`ConfigService`**: Збереження налаштувань програми.

### 🖥 Інтерфейс Користувача (`com.schoolbell.ui`)
- **`MainApp`**: Точка входу в JavaFX додаток.
- **`DashboardView`**: Головний екран з моніторингом поточного стану (таймер до дзвінка, розклад на сьогодні, візуалізація хвилини мовчання та тривог).
- **`EfirView`**: Центр керування ефіром. Перероблений у стилі "Hero Card" з анімованим пульсуючим статусом "LIVE", моніторингом підключених пристроїв у реальному часі та єдиною системою прокрутки сторінки для зручності на малих екранах.
- **`NotificationsView`**: Центр керування сповіщеннями та аудіо. Оновлений у стилі **"Modern SaaS"** з покращеним вибором пристроїв відтворення та модульною конфігурацією екстрених сигналів.
- **`SignalsView`**: Налаштування фізичних сигналів реле. Використовує сучасні картки з візуальним прев'ю ритму дзвінків (стандартний, тривога, екстрений) та вбудованою діагностикою.
- **`UIStyles`**: Централізоване сховище стилів та іконок. Включає сучасні кастомні стилі для `DatePicker` та `Spinner` (модернізовані кнопки-стрілки), а також логіку адаптивного перенесення елементів.
- **`UIComponents`**: Перевикористовувані кастомні компоненти (кнопки, картки, інпути).
- **`ScheduleEditorDialog`**: Основне вікно редагування даних з покращеною ергономікою (збільшені підписи, фіксовані колонки для запобігання обрізанню тексту). Включає:
    - `BellsEditorTab`: Налаштування часу дзвінків.
    - `WeeklyScheduleEditorTab`: Тижневий розклад з відображенням кабінетів.
    - `SubstitutionsEditorTab`: Керування замінами вчителів. Повністю локалізована система звітів та оновлений UI діалогів.
    - `ClassroomsEditorTab`: Менеджмент навчальних аудиторій.
    - `TeachersEditorTab` / `SubjectsEditorTab` / `ClassesEditorTab`: Довідники персоналу та навчальних одиниць.
    - `AnnouncementsEditorTab`: Керування оголошеннями з адаптивним дизайном тегів (FlowPane) та повною інтеграцією в систему трансляції.

### 🔌 Апаратна частина (`com.schoolbell.hardware`)
- **`RelayController`**: Взаємодія з фізичним USB-реле.

## 🛠 Поточний функціонал
1.  **Автоматичні дзвінки:** Програма вираховує час до наступної події та активує реле.
2.  **Гнучкий розклад:** Можливість швидкої зміни часу уроків та прив'язка до кабінетів.
3.  **Система замін:** Відображення актуальної інформації про заміни та формування щомісячних локалізованих звітів.
4.  **Хвилина мовчання:** Автоматичний запуск пам'ятного сигналу о 09:00 з повним візуальним супроводом на всіх екранах.
5.  **Центр керування ефіром:** Повний контроль над трансляцією з візуальною індикацією статусу, відображенням IP-адреси табло та списком підключених моніторів.
6.  **Діагностика:** Можливість тестування реле та звуку прямо з налаштувань.
7.  **Веб-дашборд:** Трансляція розкладу у реальному часі з вибором тем (`Classic`, `Modern`, `Cyber`) та підтримкою статусів безпеки.
